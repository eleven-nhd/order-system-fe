# Order System FE (Tailwind + Supabase)

Ung dung React + Vite + TypeScript de quan ly order nhom ban, thao tac truc tiep tu frontend den Supabase Postgres.

## Features

- Tab 1 - Admin: CRUD thanh vien va menu (food/drink).
- Tab 2 - Order Session: chon buyer, them dong goi mon theo tung nguoi, chot don.
- Tab 3 - Dashboard: loc ngay/tuan/thang/tat ca, xem debt matrix da can tru, xem lich su hoa don, va xoa hoa don theo bo loc.
- Tab 4 - Giai tri: thanh vien upload anh de tao bo suu tap rieng, moi nguoi deu xem duoc.
- Tab 5 - ConCac Vote: tro ly ao text co dinh de cac thanh vien vote do uong va chot 1 hoa don chung.
- Khong can backend API rieng, FE goi Supabase truc tiep bang `@supabase/supabase-js`.
- Tailwind CSS v4 cho giao dien.

## Supabase schema (PostgreSQL)

> Chay SQL nay trong Supabase SQL Editor.

```sql
create table if not exists users (
  id bigint generated by default as identity primary key,
  name text not null
);

create table if not exists menuitems (
  id bigint generated by default as identity primary key,
  name text not null,
  price numeric not null,
  type text check (type in ('food', 'drink'))
);

create table if not exists orders (
  id bigint generated by default as identity primary key,
  buyer_id bigint not null references users(id),
  order_date timestamptz not null default now(),
  total_amount numeric not null
);

create table if not exists orderdetails (
  id bigint generated by default as identity primary key,
  order_id bigint not null references orders(id) on delete cascade,
  user_id bigint not null references users(id),
  item_id bigint not null references menuitems(id),
  quantity integer not null default 1,
  price_at_time numeric not null
);

create table if not exists member_photos (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  file_path text not null unique,
  created_at timestamptz not null default now()
);
```

## Supabase storage

- Tao bucket public ten `member-photos`.
- App upload file vao bucket nay va luu metadata vao bang `member_photos`.

## RLS / Permissions

Neu FE goi truc tiep, ban can mo policy phu hop cho role `anon` hoac bat Auth + policy theo user.

Luu y: de dung tinh nang xoa hoa don theo bo loc trong Dashboard, can co quyen `delete` tren bang `orders` (va FK `orderdetails` da `on delete cascade`).

- Cach nhanh de test: tat RLS cho 4 bang tren.
- Cach dung cho production: bat RLS va viet policy chi cho phep thao tac duoc xac thuc.

## Environment variables

Tao file `.env` tu `.env.example`:

```dotenv
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-publishable-key
```

Legacy fallback (neu du an cu): `VITE_SUPABASE_ANON_KEY` van duoc ho tro.

## Run locally
